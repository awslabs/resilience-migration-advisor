⚠️ Important — Read Before Proceeding

This tool provides directional guidance to support your resilience and migration planning. By using it, you accept responsibility for validating all recommendations within your own environment. Outputs should be reviewed and discussed with your Solutions Architect, AWS partners, or internal engineering team before any action is taken.

Your environment may involve third-party dependencies — managed service providers (MSPs), SaaS platforms, partner-managed infrastructure, shared-responsibility configurations, or external APIs — that require coordination beyond what this tool covers. Identify and involve all relevant third parties, including your cloud management partner, before executing any recovery steps.

During an active incident: Check the AWS Health Dashboard and AWS Service Health Dashboard first to understand the scope of disruption before taking action.

Large-scale events: During widespread service disruptions, AWS API calls and CLI commands may experience increased latency, throttling, or intermittent timeouts due to control plane degradation. If commands appear to hang or return errors, retry with exponential backoff. Delays do not necessarily indicate incorrect commands — degraded control planes can slow responses across all AWS services. Set expectations accordingly and allow additional time for operations to complete.

⚙️ AWS CLI Retry & Resilience Best Practices

During regional impairments, AWS API calls may experience throttling, timeouts, or intermittent failures. The AWS CLI has built-in retry logic that works automatically with all commands in this tool.

Retry Modes
  • Adaptive (recommended) — standard mode plus client-side rate limiting via token bucket, dynamically adjusts call rate based on throttling responses
  • Standard (default in CLI v2) — up to 3 total call attempts with exponential backoff, max 20s
  • Legacy — up to 5 total call attempts, older retry handler with limited error coverage
Recommended Configuration
export AWS_RETRY_MODE=adaptive
export AWS_MAX_ATTEMPTS=5
  • The discovery script includes its own retry wrapper on top of CLI retries for additional resilience.
  • Use regional STS endpoints instead of the global endpoint for token operations during incidents.
  • If the console is unresponsive, try a direct regional endpoint: https://<region>.console.aws.amazon.com
  • Set AWS_RETRY_MODE=adaptive for client-side rate limiting during bulk discovery or degraded conditions.
